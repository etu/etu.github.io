## Project Overview

Personal website/blog for [elis.nu](https://elis.nu), built with Hugo and managed entirely through Nix flakes. Deployed to GitHub Pages.

## Commit Messages

Use [Conventional Commits](https://www.conventionalcommits.org/): `type(scope): description`. Common types: `feat`, `fix`, `post`, `chore`, `cron`. Examples from this repo: `post: NixOS remote unlock of encrypted ZFS over SSH`, `cron(flake): Update inputs`.

## Commands

```bash
nix develop        # Enter dev shell (or use direnv)
just run           # Local dev server at http://localhost:1313/
just compute-colors               # Recompute contrast-correct colors from config.yaml params.colors
just compute-colors themes/albatross/config.yaml  # Same for theme defaults
just update-3d-models  # Download latest 3D models release and vendor files into repo
nix build          # Production build to ./result/
nix fmt            # Format Nix files with alejandra
nix flake check    # Validate flake configuration
```

CI also runs `deadnix` and `statix` for Nix linting. There is no test suite.

## Architecture

The flake uses [numtide/blueprint](https://github.com/numtide/blueprint) ŌĆö outputs are auto-discovered from files in the repo root:
- `devshell.nix` ŌåÆ `devShells.default` ŌĆö provides `hugo` and `just`
- `packages/default.nix` ŌĆö production Hugo build; runs `hugo --minify` (3D model files are vendored)
- `packages/update-3d-models.nix` ŌĆö downloads latest `github:etu/3d-models` release, generates `content/3d-models.md` via `jq`, copies model files to `static/3d-models/`
- `packages/compute-colors.nix` ŌĆö Python script that reads `params.colors` from a Hugo config file and writes WCAG 7:1 contrast-correct hex values into `params.style`

**Hugo is pinned via nixpkgs**. No SCSS compilation ŌĆö CSS is processed as a Hugo template (`themes/albatross/assets/css/main.css`) via `resources.ExecuteAsTemplate`, injecting `site.Params.style` values. The theme and all its static dependencies are vendored directly in this repo. The only Nix inputs are `nixpkgs` and `blueprint`.

**Color system**: `config.yaml` has two sections under `params`: `colors` (human-editable base/intent colors) and `style` (auto-generated by `just compute-colors`). Edit `colors`, run `just compute-colors`, commit both. The `compute-colors` script implements the WCAG relative luminance formula and iterates to find colors meeting 7:1 contrast ratio.

**3D models** are vendored directly into the repo (`content/3d-models.md` and `static/3d-models/`). Releases from `github:etu/3d-models` are tagged `YYYY-MM-DD-<short-sha>`. Run `nix run .#update-3d-models` to pull the latest release and regenerate the vendored files, then commit the result.

## Content Structure

- `content/blog/` ŌĆö posts organized by year (`2019/` ŌĆ” `2026/`)
- `content/` ŌĆö top-level pages: `about.md`, `work.md`, `talks.md`, `cubing/`, `3d-models.md` (vendored, updated via `nix run .#update-3d-models`)
- Bilingual (English + Swedish): Swedish pages use `.sv.md` suffix or `_index.sv.md`
- `config.yaml` ŌĆö Hugo config (base URL, theme name, menus, Matomo analytics, language settings)
- `themes/albatross/` ŌĆö vendored theme including all layouts, CSS template, and static assets
